# PP
MusicLab, entrega final do projeto profissional SENAC 2025.

O MusicLab é uma plataforma digital (web) projetada para fomentar a colaboração, aprendizado e valorização artística no campo musical. Funciona como uma rede social especializada que conecta produtores, músicos, artistas e entusiastas, oferecendo um ambiente interativo para compartilhamento de projetos, samples, ideias e eventos.

Objetivo Principal: Criar um espaço de colaboração e aprendizado musical, rompendo barreiras hierárquicas e democratizando o acesso à produção musical.

Público-Alvo: Artistas independentes, produtores musicais, beatmakers, entusiastas da música, educadores musicais e ouvintes engajados.

Estado Atual: Protótipo funcional com as funcionalidades principais implementadas. Algumas funcionalidades planejadas (como Biblioteca de Samples e Sistema de Comunidades) encontram-se em fase de desenvolvimento para versões futuras.

Requisitos:
RF1:Cadastro de Usuários.	O sistema deve permitir que um novo usuário se cadastre informando nome, e-mail e nome de usuário.	Implementado;

RF2:Login de Usuários.	O sistema deve autenticar um usuário registrado via e-mail e senha.	Implementado;

RF3:Feed Interativo.	O sistema deve exibir um feed com postagens, atualizações e interações de outros usuários em tempo real.	Implementado;

RF4:Criação de Comunidades.	O usuário pode criar e gerenciar comunidades baseadas em estilos musicais ou interesses comuns.	Não implementado;

RF5:Recomendações Personalizadas.	O sistema deve sugerir músicas, artistas e projetos baseados nas preferências e comportamento do usuário.	Não implementado;

RF6:Promoção de Eventos. O usuário pode criar, visualizar e promover eventos musicais, com integração a mapas.	Parcialmente Implementado (Criação/Visualização);

RF7:Biblioteca de Samples e Tags.	O sistema deve fornecer uma biblioteca colaborativa de samples (sons, loops) com sistema de tags para busca.	Não implementado;

RF8:Chat de Mensagens Privadas.	O sistema deve permitir a troca de mensagens de texto privadas entre usuários.	Parcialmente Implementado;

RF9:Controle de Acesso por Perfil.	O sistema deve gerenciar permissões de acesso baseadas no perfil do usuário (Comum \ Administrador).	Implementado;

RF10:Fazer Postagens.	O usuário autenticado pode criar postagens de texto para compartilhar no feed.	Implementado;

RF11:	Upload de Mídia.	O usuário pode fazer upload de arquivos de áudio (MP3) e vídeo (MP4).	Implementado;

RF12:Inserir e Visualizar Eventos.	O usuário pode adicionar novos eventos e visualizar eventos existentes em uma lista ou mapa.	Parcialmente Implementado;

RF13:Reprodutor de Áudio. O sistema deve ter um reprodutor embutido para que os usuários ouçam as músicas postadas diretamente na plataforma.	Implementado;

RF14:Gestão de Projetos.	O usuário pode criar e gerenciar seus projetos musicais (álbuns, EPs, singles) dentro da plataforma. Implementado;

RF15:Ferramenta de Busca e Filtro.	O sistema deve permitir buscar e filtrar usuários, músicas, comunidades e eventos. Implementado;

Requisitos Não Funcionais (RNF):
Sistema de Gamificação	O sistema deve incorporar elementos de gamificação (pontos, níveis, conquistas) para engajar os usuários;

Segurança e Privacidade: Autenticação via JWT (JSON Web Tokens);

Usabilidade: Interface responsiva (funciona em desktop e mobile), intuitiva e de fácil aprendizado, seguindo princípios de Design Centrado no Usuário.
